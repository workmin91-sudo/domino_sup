# 재고 관리 및 자동 발주 시스템

## 개요
재고를 파악하고 재고가 부족할 때 담당 기업 직원에게 자동으로 발주서를 이메일로 전송하는 웹 기반 시스템입니다.

## 주요 기능
- 재고 데이터 입력 및 관리
- 재고 부족 시 자동 발주서 생성 및 이메일 발송
- 엑셀 파일 업로드/다운로드
- 재고 현황 조회

## 설치 방법
```bash
pip install -r requirements.txt
```

## 실행 방법
```bash
python app.py
```

## 접속 방법

### 로컬 접속
- 웹 브라우저에서 `http://localhost:5000` 접속

### 팀원 접속 (같은 네트워크)
1. 서버가 실행 중인 컴퓨터의 IP 주소 확인:
   - Windows: `ipconfig` 명령어 실행 후 IPv4 주소 확인
   - 예: `192.168.0.100`

2. 팀원들은 웹 브라우저에서 `http://[서버IP주소]:5000` 접속
   - 예: `http://192.168.0.100:5000`

3. **로그인 비밀번호: `1111`**

### 방화벽 설정
- Windows 방화벽에서 포트 5000 허용 필요
- 제어판 > Windows Defender 방화벽 > 고급 설정 > 인바운드 규칙 > 새 규칙
- 포트 선택 > TCP > 특정 로컬 포트: 5000 > 연결 허용

## 설정

### 환경 변수 설정 (보안)

시스템은 환경 변수를 사용하여 민감한 정보를 관리합니다.

#### 필수 환경 변수

다음 환경 변수들을 설정해야 합니다:

1. **EMAIL_PASSWORD**: Gmail 앱 비밀번호
   - Gmail에서 2단계 인증을 활성화합니다.
   - Google 계정 설정에서 "앱 비밀번호"를 생성합니다.
   - 생성된 앱 비밀번호를 환경 변수로 설정합니다.

2. **LOGIN_PASSWORD**: 웹 접속 비밀번호 (기본값: 1111)

#### 로컬 개발 환경

로컬에서 개발할 때는 `.env` 파일을 생성합니다:

1. `.env.example` 파일을 `.env`로 복사
2. `.env` 파일에 실제 값 입력:
   ```
   EMAIL_PASSWORD=your_gmail_app_password
   LOGIN_PASSWORD=1111
   ```

#### 배포 환경 (Vercel 등)

배포 플랫폼의 환경 변수 설정에서 다음 Key들을 추가하세요:

- `EMAIL_PASSWORD`: Gmail 앱 비밀번호
- `LOGIN_PASSWORD`: 웹 접속 비밀번호

자세한 내용은 `ENV_VARIABLES.md` 파일을 참고하세요.

### 엑셀 파일 형식

엑셀 파일 업로드 시 다음 컬럼을 포함해야 합니다:
- **품목명** (필수)
- **현재재고** (필수)
- **최소재고** (필수)
- **단위** (선택, 기본값: 개)
- **공급업체명** (선택)
- **공급업체이메일** (선택)

샘플 파일 생성:
```bash
python create_sample_excel.py
```

## 사용 방법

1. 재고 항목 추가: 웹 인터페이스에서 직접 입력하거나 엑셀 파일로 일괄 업로드
2. 재고 수정: 재고 현황 페이지에서 수정 버튼 클릭
3. 자동 발주: 재고가 최소 재고 기준 이하로 떨어지면 자동으로 이메일 발송
4. 발주 내역 조회: 발주 내역 페이지에서 모든 발주 기록 확인
